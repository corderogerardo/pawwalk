# PawWalk — Functional Review (iOS × Docker backend)

**Date:** 2026-07-01 · **Backend:** `docker compose up` in `apps/backend` (live at `http://localhost:8000`, SQLite, JWT dev secret, **no Stripe keys**).

How this was tested: the Docker backend was booted and every endpoint the iOS screens
depend on was exercised with `curl` (auth → walkers → bookings → payments → assistant,
including the user-scoping rules). Each iOS screen was read and traced to the endpoint it
calls. **Live simulator UI testing is not yet done** — that needs the app booted on a
simulator (see "Recommended tooling" at the bottom).

---

## 0. Critical bug found **and fixed** during this review (P0)

`POST /bookings`, `GET /bookings`, and `POST /payments/intent` all returned **HTTP 500**
(`sqlite3.OperationalError: no such column: bookings.user_id`). The committed dev DB
(`apps/backend/pawwalk.db`) predated the `user_id` column (plan task B11), and
`create_all()` never `ALTER`s an existing table.

- **Fixed** by deleting `pawwalk.db` and restarting so the schema was recreated + reseeded.
  All three endpoints now work (re-verified).
- **Root cause / remaining work:** the backend has **no migrations** (plan B12 Alembic is
  deferred). Until that lands, *every* future schema change silently breaks the running DB.
  → see new to-do **N1**.

## 3b. Roles, pets & walker workflow (shipped 2026-07-01)

Turned walkers into real accounts and pets into a first-class entity.

- **Roles** — `users.role` (`owner` | `walker`), chosen at sign-up. Signing up as a walker also
  creates a public walker profile (`walkers.user_id`). Role enforced server-side (`403` guards).
  Apps route on role after login: owners get the home/pets/booking flow, walkers get their own app.
- **Pets** — owner-scoped `pets` table + CRUD. Home/Profile show real pets (no more hardcoded Mochi);
  the booking form picks a pet to fill `dog_name` (contract unchanged).
- **Full walker workflow** — walker home lists **assigned walks** (`GET /bookings/assigned`) with
  **accept → start → complete** (and decline) transitions (assigned-walker + state guarded, `404`/`409`),
  plus a **profile editor** (`PATCH /walkers/me`).
- Migration `0003` (dialect-safe). **Verified:** 28 pytest green, full owner+walker+pets curl flow,
  both clients build (iOS BUILD SUCCEEDED, Android assembleDebug SUCCESSFUL) and launch.
